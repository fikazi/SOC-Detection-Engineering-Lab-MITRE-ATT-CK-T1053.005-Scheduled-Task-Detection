# Common Issues Encountered

## 1. Atomic Red Team Payload Execution Failed

Error:

```text
Access is denied
```

Cause:

- Atomic test required elevated privileges.
- Payload execution blocked.

Resolution:

- Run PowerShell as Administrator.
- Verify Atomic Red Team prerequisites.

---

## 2. Sysmon Alert Not Appearing

Cause:

- Sysmon event generated successfully.
- Detection rule did not match telemetry.

Resolution:

- Review Sysmon Event ID.
- Verify command line field mapping.
- Validate local_rules.xml syntax.

---

## 3. TheHive Alert Not Appearing

Cause:

- Wazuh alert generated successfully.
- Integration forwarding failed.

Troubleshooting performed:

- Verify API key.
- Verify integration configuration.
- Verify custom-thehive permissions.
- Verify script execution.
- Validate payload format.

Resolution:

- Fixed integration configuration.
- Successfully generated TheHive alerts.

Status:

```text
Resolved
```

---

## 4. JSON Decode Error

Error:

```text
json.decoder.JSONDecodeError
```

Cause:

- Invalid JSON format during manual testing.

Resolution:

- Validate JSON structure.
- Use proper alert payload format.

---

## 5. TheHive Dashboard Appeared Empty

Cause:

- Alerts filtered by active filters.

Resolution:

- Disable quick filters.
- Verify alert visibility settings.

Status:

```text
Resolved
```

# Lessons Learned

## Detection Engineering

- Creating alerts is easier than creating meaningful detections.
- Understanding telemetry is more important than writing rules.
- Rule tuning is necessary to reduce false positives.

---

## Threat Hunting

- Alerts should always be validated through telemetry.
- Context is critical during investigations.
- MITRE ATT&CK mapping improves analyst workflow.

---

## Incident Response

- TheHive provides structured alert tracking.
- Alert escalation requires enrichment and context.
- Investigation notes are as important as the detection itself.
